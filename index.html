Problem Statement

Manual invoice processing in mid-size companies takes 15–30 minutes per invoice, is error-prone, and lacks audit trails. Finance teams waste hundreds of hours monthly on repetitive data entry, PO matching, and reconciliation — all tasks that can be automated with AI.

Solution

Nexus-Fin automates the end-to-end AP workflow: upload a PDF invoice, extract structured data using OCR + Groq AI, perform 3-way PO matching, flag discrepancies, and push clean records directly into Google Sheets — all via a simple Streamlit UI.

Workflow

  1. Upload PDF invoice via Streamlit UI
  2. PyMuPDF extracts raw text from PDF pages
  3. Tesseract OCR handles scanned/image-based invoices
  4. Groq AI (LLaMA 3) parses structured fields: vendor, amount, line items, dates
  5. 3-way matching engine validates against PO and GRN records
  6. Discrepancies flagged with reason codes; clean records pushed to Google Sheets
  7. Approval workflow status updated in real-time dashboard

Tech Stack

Python 3.11 Streamlit Groq API (LLaMA 3) Tesseract OCR PyMuPDF Google Sheets API v4 gspread pandas

Security

Security Measures Applied

  • No API keys, credentials or secrets committed to this repository
  • All secrets managed via st.secrets (Streamlit Cloud) or environment variables
  • .gitignore configured to exclude .env files and credential files
  • No real invoice data, vendor names or financial records stored in this repo
  • Google Sheets ID injected at runtime via environment variable only
  • All sample/test data uses dummy placeholder values only
